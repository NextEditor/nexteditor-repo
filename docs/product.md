# Project Name: NextEditor
## Introduction
### Target Customer
제안서나 발표문을 작성하는 실무자, 논문을 작성하는 연구자, 리포트를 작성하는 학생 등 근거가 뒷받침되는 글을 작성하는 모든 사람

### Problem
대부분의 업무 환경에서 글을 작성할 때 자료 조사와 차트 생성의 과정이 통합되어있지 않고 비효율적이다.
- 전체 업무에서 자료 조사가 차지하는 시간이 크다
- 작업 전환에는 시간이 소요된다
- 데이터 분석에 부적절한 편향이 들어갈 수 있다

## 화면 정의

### 홈 화면
- 서비스에 처음 접속했을 떄 볼 수 있는 화면이다.
- 드라이브 서비스의 홈에 들어온 것처럼 이미 클라우드에 저장되어있는 글의 리스트 혹은 디렉토리들이 보인다.
- 마우스 우클릭 혹은 홈 화면에 있는 별도의 버튼을 통해 새로운 텍스트 파일을 생성하거나, 디렉토리(폴더)를 생성할 수 있다.
- 텍스트 파일을 더블클릭하여 글쓰기 화면으로 넘어갈 수 있다.

### 글쓰기 화면
- 기본적으로 브라우저의 전체 화면에 일정 width를 가진 텍스트 입력 칸이 생성된다.
    - 해당 입력칸은 좌우여백을 두고 있으며, 위치는 화면 중앙에서 약간 왼쪽으로 치우쳐져있다.
    - 추후에 생성될 정보/그래프 정보를 표시할 공간을 오른쪽에 남겨두기 위함이다.
- 해당 텍스트 입력칸은 markdown 문법을 지원한다.
- 글을 작성하는 데 필요한 최소한의 기능 (주석 첨부, 표 삽입)은 탑재되어 있다.

### 내보내기 (Export)
- 글쓰기 화면에서 export 버튼을 통해 해당 파일을 공유할 수 있다.
    - .pdf, .doc, .docx 파일로 변환하여 내보내기
    - 링크를 공유하여 공동작업 (editor 추가)

## 기능 정의
### LLM 활용 준비
1. 사용자가 글쓰기 화면에서 글을 작성하고 있을 때, 5초에 한 번씩 사용자가 작성하고 있는 글을 복사하여 LLM을 호출한다.
    - 호출된 LLM은 밑에서 서술한 `LLM 활용 (주장을 뒷받침하는 정보 찾기)`과 `LLM 활용 (주장을 뒷받침하는 수치 찾기)` 기능을 위해 동작한다.

    
### LLM 활용 (주장을 뒷받침하는 정보 찾기)
1. 사용자가 글쓰기 화면에서 글을 작성하고 있을 때, 5초에 한 번씩 사용자가 작성하고 있는 글을 복사하여 LLM을 호출한다. 
    - 텍스트에서 어떠한 '주장'이 담긴 문장이나 '수치'가 적힌 문장을 찾아서 반환한다.

2. 반환된 문장을 ui에서 highlight 표시한 후, 화면의 오른쪽에 로딩바를 띄운다.
    - 화면 오른쪽의 로딩 바는 highlight 표시된 문장과 비슷한 y좌표에 존재해야 하며, 일정 크기의 사각 박스 안에 존재해야 한다.
        - 해당 사각 박스의 우측 상단에는 X 표시가 있으며, 해당 버튼을 누를 경우 해당 문장을 수정하기 전까지는 해당 문장에서 다시 LLM을 호출하지 않는다.
    - 이 때 또다른 LLM을 호출한다.
    - highlight 표시된 문장을 기준으로 앞 500글자, 뒷 500글자를 같이 복사하여 LLM에게 전달하고, 해당 문단을 가장 잘 설명할 수 있는 뉴스, 논문 등을 찾아달라고 요청한다.
        - 만약 앞이나 뒤에 500글자만큼 복사할 내용이 없을 경우 보낼 수 있을 만큼만 전송한다.

3. 정보를 찾을 때는 논문, 공신력 있는 저널에 실린 글, 뉴스 기사 등 공신력 있는 출처를 중심으로 검색한다.

4. 만약 정보를 발견했다면, 화면 오른쪽의 로딩 바를 삭제하고 해당 정보를 띄워준다.
    - 일정 크기의 사각 박스를 넘어서는 정보량은 `...` 표시를 통해 조정한다.
    - 정보가 1개 이상일 경우 리스트 형식으로 제공한다.
    - 해당 정보는 클릭가능하며, 클릭할 경우 새로운 탭이 열리며 해당 정보를 가져온 페이지로 이동한다.
    - 각각의 리스트의 오른편에는 X, O 표시가 각각 있으며, X표시를 누르게 되면 리스트에서 사라진다.
        - O표시를 누르면 다른 모든 리스트가 사라지며, 해당 페이지가 주석으로 이동한다.
