## Find reason
1. 정보를 찾는 중이라는 인식 제공하기 (우선순위 Medium)
    - 사용자 입장에서는 문장을 작성하고나서 정보를 찾고있는건지 아닌지 잘 모른다.
        - 노란색으로 Highlight 표시해두고, 오른쪽의 Recommendation 탭에 사각형과 로딩바 필요

2. 근거를 추가하는 로직 변경하기 (우선순위 High)
    - 지금은 어떤 방식으로 동작하는진 모르겠는데 현재는 아래와 같은 로직인 것같음:
        - (마침표를 기준으로) 문장 인식 -> 근거 찾기 -> 근거 리턴 -> 문서에 추가 버튼 클릭 -> 해당 페이지 읽어서 관련 문장 자동으로 추가
    - 그런데 글을 작성하는 사용자 입장에서 글이 이미 많이 작성되어있을텐데, 자동으로 글이 추가되면 사용자가 알아보기 어려울 수 있다. (원하지 않을수도 있음)
        - 그냥 주석만 추가하는게 어떨까?
        - 아니면 노션처럼 AI가 추가한 글을 파란색으로 highlight하고, 사용자가 추가할지 말지를 결정하는게 좋을 것같음
        - 일단 주석은 반드시 필요한듯

## Home tab
1. 홈 화면 만들기 (우선순위 Low)
    - 사용자는 여러 개의 문서를 생성할 수 있다. 따라서 로그인 시 글을 작성하는 공간이 아니라 드라이브와 같은 화면 (만들어놓은 글을 볼 수 있는 드라이브 공간)으로 가야한다.

## Analyze data
1. 정보를 찾을 때 raw data도 같이 찾아보라고 해야할듯 (우선순위 VERY HIGH)
    - 그래서 raw data와 reason을 구분해서 Recommendation tab에 표시해서 보여줘야 할듯
    - 만약 사용자가 raw data 여러개 중 하나를 고르면,
        1) 해당 페이지에서 raw data 추출 (다운로드)
        2) 데이터 분석 (차트 생성, python으로 작성하고 실행한 결과 이미지 추출)
        3) 인사이트 작성 (해당 차트로 어떤 사실을 알 수 있는지)
        4) 사용자 리턴 (이미지+인사이트 세트 여러개)
    - 근데 이걸 구현하려면 어떻게 해야할까? 위 작업을 수행하는 Container를 하나 띄워야하나?

2. 분석 결과를 ui적으로 어떻게 보기 쉽게 만들어줄 수 있을까? (우선순위 VERY HIGH)
    - 현재 raw data 분석이 진행되면 (이미지+인사이트) 세트가 여러개 리턴된다
    - 이 중에서 사용자가 사용하고싶은 세트가 다를텐데, 어떻게 선택하게 할 수 있을까?
    - 사용자가 특정 데이터를 선택하면, 해당 데이터가 있는 Recommendation 박스가 잠시 비활성화된다.('데이터 분석 중...' 표시와 함께)
    - 그동안 사용자는 글을 마저 쓰거나, 다른 데이터를 분석하거나 등등 활동을 진행할 수 있음
    - 분석이 완료되면, 해당 Recommendation 박스가 주황색 (혹은 파스텔톤 빨간색)으로 바뀌면서 '분석 완료!'로 바뀜
        - 사용자가 해당 버튼을 클릭하면 모달이 하냐 띄워지고, 해당 모달 안에 리턴받은 그래프 이미지와 인사이트가 존재한다.
    - 복수 개의 이미지를 선택하여 글에 삽입할 수 있다. 해당 사진에 대한 주석으로 데이터 출처가 적힌다.
